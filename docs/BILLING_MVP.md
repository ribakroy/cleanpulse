# CleanPulse Billing MVP

עבור גרסת ה-MVP של CleanPulse, מודל הגבייה ותוכניות המנוי מנוהל באופן ידני על ידי מנהל העל (Super Admin) מתוך ה-Owner Console, ללא חיבור ישיר לסליקה או מערכות חיצוניות כגון Stripe בשלב זה.

## 1. הגדרות תוכניות המנוי

המערכת תומכת בתוכניות המנוי הבאות:

| שם תוכנית | תיאור | מחיר מומלץ |
| :--- | :--- | :--- |
| **Free** | תוכנית חינמית מוגבלת למסך אחד בלבד | 0 ₪ |
| **Starter** | חבילה ראשונית לעסקים קטנים (עד 3 מסכים) | 99 ₪ |
| **Basic** | החבילה הסטנדרטית (עד 5 מסכים) | 199 ₪ |
| **Pro** | חבילה מורחבת (עד 15 מסכים ודוחות מתקדמים) | 399 ₪ |
| **Enterprise** | חבילה מותאמת אישית ללא הגבלת מסכים | מותאם אישית |
| **Demo** | מיועד לחשבונות הדגמה פנימיים | 0 ₪ |

---

## 2. שדות המודל (SaaS Schema Extension)

שדות ה-SaaS והחיוב נוספו לרשומת ה-`OrganizationRecord` בצורה אופציונלית לשמירה על תאימות נתונים:

- `status` (`"active" | "trial" | "suspended" | "cancelled"`): סטטוס המנוי בפועל.
- `billingStatus` (`"active" | "trialing" | "past_due" | "cancelled" | "manual"`): סטטוס הגבייה לצורך מעקב.
- `billingEmail`: כתובת האימייל אליה יש לשלוח דרישות תשלום וחשבוניות.
- `companyName`: שם התאגיד/החברה החוקית (במידה ושונה משם המותג).
- `contactName` ו-`contactPhone`: פרטי איש הקשר הפיננסי לתיאום תשלומים.
- `allowedScreensLimit`: מגבלת כמות המסכים הפעילים המותרת לארגון.
- `monthlyPrice`: מחיר המנוי החודשי בשקלים חדשים (₪).
- `notes`: הערות פנימיות של הבעלים (הסדרי תשלום מיוחדים, הנחות וכדומה).

---

## 3. תזרים ניהול המעקב (Manual Billing Workflow)

1. **הקמת לקוח**: בעת הקמת עסק חדש ב-`/super/organizations/new`, מגדירים את ה-allowedScreensLimit, ה-monthlyPrice ופרטי הקשר לגבייה.
2. **תקופת ניסיון (Trial)**: כברירת מחדל הלקוח מתחיל בסטטוס `trial`. מנהל העל יכול להגדיר תאריך סיום או להעבירו ידנית למצב פעיל לאחר הסדר התשלום הראשון.
3. **מעקב MRR**: בעמוד `/super/billing` מחושב סכום ה-MRR (הכנסה חודשית חוזרת) בזמן אמת על בסיס סכום העלויות של כל הלקוחות הפעילים.
4. **טיפול בפיגורים (Overdue / Past Due)**: במידה ולקוח לא שילם, מנהל העל מעדכן את סטטוס הגבייה שלו ל-`past_due`. לקוחות אלו יקבלו אזהרה בולטת בדשבורד של מנהל העל.
5. **השעיית לקוח (Suspension)**: במידה ונדרש לחסום גישה ללקוח עקב אי-תשלום, מנהל העל מעביר את הסטטוס ל-`suspended`. מצב זה נועל מיידית את גישת הניהול של העסק ב-`/admin` ומציג למשתמשיו הודעה ברורה בעברית שעליהם להסדיר את התשלום מול הבעלים.
