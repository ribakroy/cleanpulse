# CleanPulse Super Admin Console

אזור ניהול העל (Owner Console) הוא דשבורד פנימי המיועד לבעלי ומפעילי מערכת CleanPulse לניהול מקיף של כל הארגונים והלקוחות, פיקוח על מדדי שימוש, גבייה ידנית ופעילות המערכת.

## 1. גישה ואבטחה

- **תפקיד מוגן**: רק משתמשים בעלי תפקיד `super_admin` מורשים לגשת לנתיבי `/super`.
- **חסימת גישה**: משתמשים רגילים של עסקים (כגון `owner`, `admin`, `manager`, `cleaner`) שמנסים לגשת ל-`/super` יופנו אוטומטית חזרה ל-`/admin/dashboard` של העסק שלהם.
- **הפניה בכניסה (Login)**: מנגנון הכניסה מזהה את תפקיד המשתמש. מנהלי על מופנים ישירות ל-`/super/dashboard`.
- **ארגון המערכת**: משתמשי `super_admin` משויכים לארגון מערכת וירטואלי מיוחד בשם `org_system`.

## 2. יצירת Super Admin ראשון

כדי להתחיל לעבוד, יש ליצור משתמש סופר-אדמין ראשון. הפעולה מתבצעת בצורה בטוחה מחוץ לממשק על ידי הרצת סקריפט ה-CLI הייעודי.

### הרצה בסביבה מקומית (Local):
בסביבה מקומית, הסקריפט יכתוב ישירות ל-`data-local/data/`:
```bash
npm run create-super-user [אימייל] [סיסמה] [שם מלא]
```

אם לא יועברו פרמטרים, ייווצר משתמש ברירת המחדל הבא:
- **אימייל**: `superadmin@cleanpulse.app`
- **סיסמה**: `SuperAdmin123456!`
- **שם**: `מנהל על מערכת`

### הרצה בסביבת פרודקשן (Production / GitHub Data Repository):

> [!IMPORTANT]
> - בסביבת פרודקשן (כאשר `DATA_ADAPTER=github`), הסקריפט **חוסם לחלוטין** את השימוש בפרטי ברירת המחדל הבסיסיים מחשש לאבטחה.
> - עליך לספק אימייל, סיסמה מורכבת ושם מלא כפרמטרים מפורשים בריצה.
> - הסיסמה חייבת לעמוד בדרישות אבטחה: לפחות 8 תווים, אות גדולה באנגלית, אות קטנה ומספר.
> - הסיסמה לא תודפס בלוגים של הריצה כדי לשמור על סודיות.
> - **אזהרה**: אין להריץ את ה-Seed המלא (`seed:github`) שכן הוא ימחק ויחליף את כל מאגר הנתונים הקיים. הסקריפט `create-super-user` שומר על הנתונים הקיימים ורק מוסיף או מעדכן את המשתמש הניהולי המבוקש.

להרצה בטוחה בפרודקשן מול מאגר ה-GitHub, השתמש בפקודה הבאה והעבר את משתני הסביבה והפרמטרים המתאימים:
```bash
DATA_ADAPTER=github \
GITHUB_DATA_OWNER=ribakroy \
GITHUB_DATA_REPO=cleanpulse-data \
GITHUB_DATA_BRANCH=main \
GITHUB_DATA_TOKEN=your_github_token_here \
npm run create-super-user "email@domain.com" "StrongPassword123!" "שם מנהל על"
```

## 3. מבנה ה-Routes

כל נתיבי ה-Super Admin ממוקמים תחת הסיומת `/super` ומוגנים על ידי ה-layout הראשי:

- **`/super/dashboard`**: לוח בקרה עם KPIs כלליים, עסקים בעייתיים הדורשים טיפול ופעילות אחרונה.
- **`/super/organizations`**: רשימת כל העסקים, מנועי סינון וחיפוש.
- **`/super/organizations/new`**: טופס הקמת עסק חדש ויצירת ה-Owner הראשי עם סיסמה זמנית.
- **`/super/organizations/[id]`**: כרטיס לקוח מורחב, עריכת פרטי קשר ו-SaaS, שינוי סטטוס (פעיל / מושעה / מבוטל), הוספת משתמשים ואיפוס סיסמה.
- **`/super/billing`**: מרכז גבייה ידני וניהול תוכניות, מחירים וסיכום MRR.
- **`/super/usage`**: ניתוח מדדי שימוש וחומרה (מסכים לא מקוונים, כמות דיווחים).
- **`/super/system`**: תקינות שרת, חיבור לגיטהאב, ספק מייל וספירת רשומות במאגר.
- **`/super/activity`**: יומן פעילות ניהולית מלא.

## 4. שפת עיצוב וחוויית משתמש (UI/UX)

- **מראה מובחן**: ה-Shell של ה-Super Admin מעוצב בצבע כחול כהה/Navy עמוק (`bg-slate-900` ו-`bg-slate-950`) בשילוב כחול מותג ו-Sky Blue, כדי להעניק למפעיל תחושה ברורה של "מרכז שליטה פנימי" שונה מהאדמין הרגיל של העסק.
- **RTL ועברית מלאה**: תמיכה מלאה ביישור ימין ושפה עברית נקייה ללא מונחים טכניים מורכבים בממשק עצמו.
